# Criteria: Weight Rules

定義高 / 中 / 低權重的分類原則。供 AI 在評估時確認各項目的權重是否合理。

## 權重分類原則

| 權重 | 條件 |
|------|------|
| **高** | 缺失會直接導致後續步驟失效、產出錯誤，或違反安全/合規要求 |
| **中** | 缺失會降低品質或造成後續工作量增加，但不阻斷主流程 |
| **低** | 缺失是最佳實踐的缺席，不影響當前功能正確性 |

## 各層的高權重範例

| 層 | 高權重項目 |
|----|---------|
| spec | 驗收標準缺失（沒有就沒有測試依據）|
| scenarios | Happy path 缺失（基本流程都沒有）|
| test-cases | 無唯一 ID（無法追蹤和對應）、無預期結果（無法驗收）|
| automation | 硬寫憑證（安全違規）、使用 data-testid（SIT 必定失敗）|
| report | Pass/Fail 不來自 execution-results（AI 捏造結果）|

## 不可降低權重的情況

以下項目即使使用者要求略過，也**不可降低為中/低權重**：

- automation 的硬寫憑證或 URL
- report 的 Pass/Fail 來源
- spec 的驗收標準
